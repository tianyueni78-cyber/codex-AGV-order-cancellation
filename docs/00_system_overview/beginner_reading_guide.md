# 项目易读版：这套代码到底在做什么

## 先用一句话理解

这个项目是在研究：

> 怎么安排工件、机器和 AGV 小车，让生产尽量快，同时尽量省电。

它不是单纯让机器加工，也不是单纯让 AGV 搬运，而是把两件事放在一起考虑：

- 工件要在机器上加工。
- AGV 要把工件从一个地方搬到另一个地方。
- 机器和 AGV 都会消耗能量。
- AGV 电量不够时还要充电。

所以这个项目的核心是：

```text
排产 + 搬运 + 能耗 + 多目标优化
```

## 代码大概怎么跑起来

可以先不要管复杂算法，只记住这条主线：

```text
读数据
-> 生成一批可能的调度方案
-> 把每个方案真正排成时间表
-> 计算每个方案用了多久、耗了多少电
-> 算法保留更好的方案，继续生成新方案
```

对应到代码，大概是：

```text
read_fjsp / Excel 数据
-> init.m
-> sorting.m
-> fitness.m
-> NSGA2.m / variation.m
```

## 这些名字到底是什么意思

### 数据层：先把题目读进来

系统要先知道“要调度什么”。

主要数据包括：

- `.fjs` 文件：工件、工序、可选机器、加工时间。
- 机器 Excel：机器距离、机器能耗。
- AGV Excel：AGV 空载/负载能耗。
- 实验参数：种群规模、迭代次数、交叉概率、变异概率等。

可以理解为：

```text
数据层 = 给算法出题
```

相关文件：

```text
src/data/read_fjsp.m
raw_code/benchmarkRead.m
raw_code/dif_main.m
raw_code/same_main.m
```

### 编码层：把调度方案写成一串数字

算法不能直接操作真实工厂，所以代码把一个调度方案写成一条“染色体”。

染色体分成四段：

| 名字 | 中文理解 | 它决定什么 |
|---|---|---|
| OS | 工序顺序 | 哪些工序先安排 |
| MS | 机器选择 | 每道工序去哪台机器加工 |
| AS | AGV 选择 | 每道工序由哪辆小车搬 |
| SS | 速度选择 | AGV 空载/负载时用哪个速度 |

可以理解为：

```text
编码层 = 把一个调度方案压缩成一串数字
```

相关文件：

```text
raw_code/init.m
raw_code/NSGA-II/variation.m
```

### 解码层：把一串数字变成真实时间表

这是项目里非常关键的一步。

`sorting.m` 会把染色体解释成真实调度过程，比如：

- 哪个工件先加工
- 选哪台机器
- 哪辆 AGV 去搬
- AGV 什么时候空载移动
- AGV 什么时候负载移动
- 机器什么时候开始加工
- 机器什么时候结束
- AGV 什么时候需要充电
- 工件最后什么时候到卸载站

可以理解为：

```text
sorting.m = 把方案真正排成时间表
```

相关文件：

```text
raw_code/INSGA-II/sorting.m
raw_code/NSGA-II/sorting.m
raw_code/MOEAD/sorting.m
```

### 评价层：判断这个方案好不好

`fitness.m` 会根据 `sorting.m` 排出来的时间表，计算两个核心指标：

1. 最大完工时间  
   所有工件都加工完并送到卸载站，最晚完成的那个时间。

2. 总能耗  
   包括机器加工能耗、机器空闲能耗、AGV 运输能耗。

可以理解为：

```text
fitness.m = 给调度方案打分
```

相关文件：

```text
raw_code/INSGA-II/fitness.m
raw_code/NSGA-II/fitness.m
raw_code/MOEAD/fitness.m
```

### 搜索层：算法继续找更好的方案

算法会不断尝试新方案。

过程大概是：

```text
先生成一批方案
-> 给每个方案打分
-> 保留比较好的
-> 交叉、变异生成新方案
-> 再打分
-> 继续迭代
```

可以理解为：

```text
搜索层 = 不断试方案，留下更好的
```

相关文件：

```text
raw_code/NSGA-II/NSGA2.m
raw_code/NSGA-II/variation.m
raw_code/NSGA-II/non_domination.m
raw_code/NSGA-II/replace_chrom.m
```

## 为什么不是找一个唯一最优解

这个项目有两个目标：

```text
时间尽量短
能耗尽量低
```

但这两个目标经常冲突：

- AGV 跑快一点，可能时间短，但更耗电。
- AGV 跑慢一点，可能省电，但工件等待更久。
- 选加工快的机器，可能能缩短时间，但机器能耗更高。

所以算法最后找的不是一个唯一答案，而是一组折中方案。

这组方案叫：

```text
Pareto 解集
```

你可以把它理解成：

```text
一组“各有优点、互相不好直接替代”的方案
```

## 第一次读代码建议顺序

如果你第一次读，不建议从所有算法一起看。

建议顺序是：

```text
1. src/data/read_fjsp.m
   先看 .fjs 数据怎么读进来。

2. raw_code/init.m
   再看初始染色体怎么生成。

3. raw_code/INSGA-II/sorting.m
   重点看染色体怎么变成真实调度。

4. raw_code/INSGA-II/fitness.m
   看方案怎么被计算完工时间和能耗。

5. raw_code/NSGA-II/NSGA2.m
   最后看算法怎么反复生成和筛选方案。
```

## 最小心智模型

先记住这张图就够了：

```text
.fjs / Excel
   ↓
读入工件、机器、AGV、距离、能耗
   ↓
init.m 生成染色体
   ↓
sorting.m 排成机器和 AGV 时间表
   ↓
fitness.m 算完工时间和总能耗
   ↓
NSGA2.m 继续找更好的染色体
```

## 最重要的几个文件

| 文件 | 你可以先怎么理解 |
|---|---|
| `read_fjsp.m` | 读取标准算例 |
| `init.m` | 先造一批调度方案 |
| `sorting.m` | 把方案排成真实时间表 |
| `fitness.m` | 算这个方案好不好 |
| `variation.m` | 通过交叉、变异生成新方案 |
| `NSGA2.m` | 控制算法一代一代搜索 |

## 一句话总结

这个项目最核心的流程是：

```text
算法不断尝试不同排产方案，
系统把每个方案排成真实时间表，
再用完工时间和能耗来判断方案好不好。
```

如果只记一个重点，就是：

> 算法搜索的是染色体，`sorting.m` 把染色体变成调度，`fitness.m` 判断这个调度好不好。
